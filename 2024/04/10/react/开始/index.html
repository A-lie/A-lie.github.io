class 组件与类组件

在 React 中，”class 组件”和”函数组件”（有时也称为”函数式组件”）是两种主要的组件类型，而”类组件”是指使用 ES6 类语法定义的 React 组件，可以包含状态（state）和生命周期方法。在 React 16.8 版本之后，引入了 Hooks，使得函数组件也能拥有状态和其他类组件的特性，因此函数组件也可以称为”函数式组件”。
以下是”类组件”和”函数组件”之间的一些不同之处：

类组件（Class Components）：
1、定义方式：使用 ES6 类语法定义组件，继承自 React.Component 类。
2、状态管理：可以使用 this.state 来管理组件的状态，同时可以使用 this.setState 来更新状态。
3、生命周期方法：可以使用生命周期方法（如 componentDidMount、componentDidUpdate 等）来处理组件的生命周期事件。
4、实例化：每个类组件都有自己的实例，可以通过 this 访问组件实例的属性和方法。
5、性能优化：可以使用 shouldComponentUpdate 来优化组件的更新性能。

函数组件（Function Components）：
1、定义方式：使用函数声明定义组件，接收 props 作为参数。
2、状态管理：使用 Hooks（如 useState）来管理组件的状态。
3、生命周期：使用 Hooks（如 useEffect）来处理组件的生命周期事件。
4、无实例：函数组件是无状态的，没有实例，因此无法通过 this 访问实例属性和方法。
5、简洁性：通常比类组件更简洁和易于阅读，尤其是对于只需要渲染 UI 的简单组件。

随着 Hooks 的引入，函数组件在很大程度上取代了类组件的地位，因为函数组件具有更简洁的语法和更好的性能。通常情况下，推荐使用函数组件来编写新的 React 组件，除非需要使用类组件特有的功能（如生命周期方法）或者在现有项目中需要与类组件进行交互。
