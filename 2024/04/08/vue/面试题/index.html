vue 中的 spa 应用如何优化首屏加载速度?

1、懒加载
2、预渲染：由于浏览器在渲染出页面之前，需要先加载和解析相应的 html、css 和 js 文件，为此会有一段白屏的时间，可以添加 loading，或者骨架屏幕尽可能的减少白屏对用户的影响体积优化
3、合理使用第三方库：对于一些第三方 ui 框架、类库，尽量使用按需加载，减少打包体积
4、提高代码使用率：利用代码分割，将脚本中无需立即调用的代码在代码构建时转变为异步加载的过程
5、封装：按照项目需求进行全局组件，插件，过滤器，指令，utils 等做一些公共封装，可以有效减少我们的代码量，而且更容易维护资源优化
6、图片懒加载：使用图片懒加载可以优化同一时间减少 http 请求开销，避免显示图片导致的画面抖动，提高用户体验
7、使用 svg 图标：相对于用一张图片来表示图标，svg 拥有更好的图片质量，体积更小，并且不需要开启额外的 http 请求
8、压缩图片：可以使用 image-webpack-loader，在用户肉眼分辨不清的情况下一定程度上压缩图片

Vue 中的 Key 的作用是什么

key 的作用主要是为了高效的更新虚拟 DOM。另外 vue 中在使用相同标签名元素的过渡切换时，也会使用到 key 属性，
其目的也是为了让 vue 可以区分它们，否则 vue 只会替换其内部属性而不会触发过渡效果。

组件中写 name 选项有哪些好处

1、可以通过名字找到对应的组件（ 递归组件：组件自身调用自身 ）
2、可以通过 name 属性实现缓存功能（keep-alive）
3、可以通过 name 来识别组件（跨级组件通信时非常重要）
4、使用 vue-devtools 调试工具里显示的组见名称是由 vue 中组件 name 决定的

怎么理解 vue 的生命周期?

Vue 的生命周期指的是 Vue 实例从创建到销毁的整个过程中经历的一系列阶段。理解 Vue 的生命周期可以帮助开发者在不同阶段执行相应的操作，例如初始化数据、监听数据变化、销毁事件等。

vue 双向数据绑定原理

双向数据绑定采用数据劫持结合发布者-订阅者模式的方法。通过 object.defindeProperty()方法来劫持各个属性的 set 和 get，在数据变动时发布消息给订阅者，触发相应的监听回调。（addEventListener ）

说一下对 Vue2.x 响应式原理的理解

Vue 在初始化数据时，会使用 Object.defineProperty 重新定义 data 中的所有属性，当页面使用对应属性时，首先会进行依赖收集(收集当前组件的 watcher)，如果属性发生变化会通知相关依赖进行更新操作(发布订阅)。

Proxy 相比 defineProperty 的优势在哪里

Vue3.x 改用 Proxy 替代 Object.defineProperty

原因在于 Object.defineProperty 本身存在的一些问题：
Object.defineProperty 只能劫持对象属性的 getter 和 setter 方法。
Object.definedProperty 不支持数组(可以监听数组,不过数组方法无法监听自己重写)，更准确的说是不支持数组的各种 API(所以 Vue
重写了数组方法。

而相比 Object.defineProperty，Proxy 的优点在于：
Proxy 是直接代理劫持整个对象。
Proxy 可以直接监听对象和数组的变化，并且有多达 13 种拦截方法。

目前，Object.definedProperty 唯一比 Proxy 好的一点就是兼容性，不过 Proxy 新标准也受到浏览器厂商重点持续的性能优化当中。

vueRouter 有哪几种导航守卫？

1、全局前置/钩子：beforeEach、afterEach
2、路由独享的守卫：beforeEnter
3、组件内的守卫：beforeRouteEnter、beforeRouteUpdate、beforeRouteLeave

watch 与 computed 的区别是什么？

1、都是观察数据变化的（相同）
2、computed 有缓存，它依赖的值变了才会重新计算，watch 没有；
3、watch 支持异步，computed 不支持；
4、watch 是一对多（监听某一个值变化，执行对应操作）；computed 是多对一（监听属性依赖于其他属性）
5、watch 监听函数接收两个参数，第一个是最新值，第二个是输入之前的值；
6、computed 属性是函数时，都有 get 和 set 方法，默认走 get 方法，get 必须有返回值（return）

watch 的 参数：
deep：深度监听
immediate ：组件加载立即触发回调函数执行

说一下 v-if 与 v-show 的区别

1、手段
v-if 是动态的向 DOM 树内添加或者删除 DOM 元素
v-show 是通过设置 DOM 元素的 display 样式属性控制显隐
2、编译条件
v-if 是惰性的，如果初始条件为假，则什么也不做。只有在条件第一次变为真时才开始局部编译
v-show 是在任何条件下(首次条件是否为真)都被编译，然后被缓存，而且 DOM 元素保留
3、性能消耗
v-if 有更高的切换消耗
v-show 有更高的初始渲染消耗
4、使用场景
v-if 适合运营条件不大可能改变
v-show 适合频繁切换

什么是闭包？

闭包：引用父级函数中的变量就会产生闭包
作用：1、延伸了函数的作用范围，读取函数内部的变量 2、让变量的值始终保持在内存中，调用后不会被回收。3、方便调用上下文的局部变量，利于代码封装。
容易引起的问题：内存泄露

在项目中，如何处理大规模数据的渲染和性能优化?

1.延迟加载:如果数据量很大，可以采用延迟加载的方式，根据用户的滚动或请求来动态加载数据，而不是一次性加载全部数据。这可以减少初始加载时间和内存占用。 2.分批次渲染:将大规模数据分为小的批次进行渲染，可以使用分页或滚动加载的方式。只更新当前可见区域的数据，而不是整个数据集，以提高渲染性能。 3.虚拟化列表:对于长列表或表格，可以采用虚拟化列表技术。只渲染当前可见区域的内容，并根据滚动位置动态加载和卸载数据项。这可以显著提升性能，特别是当数据量非常大时。 4.数据压缩和缓存:对于大规模数据，可以尝试使用数据压缩算法来减少数据量。同时，使用缓存技术可以在后续访问时减少对数据源的请求，提高渲染性能。
5.web workers:利用浏览器的 Web workers 技术，在后台线程中处理数据，避免阻塞主线程的渲染操作。这可以提高用户界面的响应性能。 6.内存优化:对于大规模数据，内存占用可能是一个问题。可以通过优化数据结构、及时释放不再需要的资源、使用内存分页等方式来减少内存占用。
7.CPU 优化:对于复杂操作或计算密集型任务，可以优化算法和数据处理过程，以减少 CPU 的使用量。

mvvm 与 mvc 的区别：

1、mvvm 各部分的通信是双向的，而 mvc 各部分通信是单向的；
2、mvvm 是真正将页面与数据逻辑分离放到 js 里去实现，而 mvc 里面未分离。

vue3 为什么用 setup 函数

语法糖、可以不需要写 data, methods 哪些、setup 方法执行的时机比 beforeCreate 更早

垃圾回收机制：

1:引用计数法
2:标记清除法(从根对象(也就是 window)开始查找所有引用的对象，并标记为‘使用中’，没有标记为使用中的对象就是垃圾对象)

JS 中原型的理解

有 2 个原型(prototype:是显示原型,给程序员使用,存在于函数中。proto:是隐式原型，给浏览器使用，存在于实例对象中。)
产生时机: 显示原型是在函数创建的时候产生的，里面有显示原型和隐示原型。 隐示原型是在 new 或者可以说是创建实例对象的时候产生的。
原型的作用:
为了实现数据共享，节省内存空间;
实现继承，其实也是为了数据共享
原型链:原型链指的是隐示原型和隐示原型的关系【其作用是为了查找对象中的属性或方法】，所以也 叫做隐示原型链

作用域：

作用:隔离变量，避免数据污染。
分类:全局作用域，函数作用域(局部作用域)、块级作用域(es6 中 let const 声明变量所在的区域–>花括号{}区域:比如:if{}、 循环{}等)

nextTick 的理解：

在数据更新之后，立刻调用这个方法，获取更新后的 DOM

浏览器的缓存有哪些?

图片，字体，本地存储（localStrong、sessionStrong）,cookie 缓存
