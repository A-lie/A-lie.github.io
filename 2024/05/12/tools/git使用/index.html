git代码提交流程

git仓库分为工作区、暂存区、本地仓库、远程仓库。

  1. 工作区为当前修改的文件
  2. 暂存区为等待提交本地仓库的文件，通过git add {filepath}来将修改的文件保存到暂存区
  3. 通过git commit -m "xxx"提交到本地仓库一定是暂存区的文件
  4. 提交到本地仓库后，为了同步到托管平台，需要执行git push将代码推送到远程托管平台

  1. 查看提交日志

冲突管理

当远程托管平台的仓库已经被别人提交了，自己也提交了本地仓库，执行git pull的时候会报错merge conflict即合并冲突。

  1. 执行git pull的时候，git已经尝试帮你去自动merge合并了，如果可以自动合并，会需要你提交合并的提示文本，可以直接用默认的，只需要退出这个文本即可
